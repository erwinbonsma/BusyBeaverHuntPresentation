Turing Machines

Busy Beaver Problem

For a given \(n\), find the \(n\)-state binary-symbol Turing machine that:

  • eventually halts and
  • produces the most 1's on tape

This number is called \( \Sigma (n) \)

Busy Beaver Problem (2)

For a given \(n\), find the \(n\)-state binary-symbol Turing machine that:

  • eventually halts and
  • has executed the most instructions

This number is called \( S(n) \)

Busy Beaver functions

The \( \Sigma(n) \) and \( S(n) \) Busy Beaver functions are:

  • well-defined,
  • non-computable, and
  • asymptotically larger than any computable function

Tibor Radó, "On Non-Computable Functions", 1962

Halting Problem

The reason both \( \Sigma(n) \) and \( S(n) \) are non-computable is because of the Halting Problem:

There is no general algorithm for deciding that a given Turing Machine halts

Alan Turing, "On Computable Numbers, with an Application to the [Decision Problem]", 1936

Asymptotically larger

The function \(f(x)\) is asymptotically larger than \(g(x)\) if there is a value \(x_{0}\) so that: \[\forall x: x > x_{0} \implies f(x) > g(x)\]

2L programming language

an esoteric language by Gregor Richards

  • 2D program grid
  • Two programming symbols: DATA and TURN
  • 1D data "tape" storing integers

Example program

Simple loop

2L Busy Beaver problem

For a given \(n\), find the \(n \times n\) 2L program that:

  • eventually halts and
  • has executed the most steps

Hand-crafted 9x9 program

Search challenges

  • For an \(n \times n\) grid there are \( 3^{n^{2}} \) possible programs
  • Program execution time increases rapidly for increasing grid sizes

Search optimization

Ignore Don't Cares

  • For a given program not all grid positions are visited
  • These "Don't Care" locations therefore do not need to be instantiated
